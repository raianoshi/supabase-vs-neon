🧠 What Each Folder Means

/app Routing

This is your website pages. Every folder = a URL.

Example:

/profile/john → app/profile/[username]/page.tsx

/components UI

Reusable visual parts like buttons, timers, cards.

Example component:

export function Button({ children }) {
  return (
    <button className="px-4 py-2 bg-blue-500 text-white">
      {children}
    </button>
  );
}

/services Logic

All business rules live here (NOT in UI).

Example:

export async function startPractice(userId, exerciseId) {
  return db.practice_sessions.create({
    user_id: userId,
    exercise_id: exerciseId,
    started_at: new Date()
  });
}

/lib Core tools

Database, Stripe, authentication helpers.

export function requireUser(session) {
  if (!session) throw new Error("Not authenticated");
}

🎯 Mental Model

Think of it like this:

  • /app → pages (what user sees)
  • /components → LEGO pieces (UI blocks)
  • /services → brain (rules & logic)
  • /lib → tools (Stripe, DB, auth)

⚡ Example Flow (Practice Session)

User clicks “Start Exercise”
  ↓
app/practice/page.tsx
  ↓
calls services/practice.service.ts
  ↓
writes to DB (lib/db.ts)
  ↓
UI updates via components/Timer.tsx

🚀 Why This Structure Works

  • Keeps UI clean and simple
  • All logic is reusable
  • Easy to scale social + payments
  • Prevents messy Next.js spaghetti code
